# Watch Anime — Design Findings

## Problem Statement

Build a personal anime streaming website that can browse, search, and stream anime episodes with watchlist and progress tracking.

## Key Decisions

1. **Content source:** Self-hosted Consumet API (Node.js) — aggregates from Gogoanime, Zoro, etc.
2. **Architecture:** Laravel as a thin proxy/cache layer between React frontend and Consumet API. All Consumet calls go through Laravel controllers.
3. **Stack:** Laravel 12 + React 19 + Inertia.js + TypeScript + Tailwind CSS (via Laravel Breeze)
4. **Database:** SQLite — simple, no extra server for a hobby project
5. **Cache:** Laravel file cache — anime metadata 24h, episode lists 6h, streaming links 30min
6. **Video player:** ArtPlayer — HLS support, subtitles, picture-in-picture
7. **Auth:** Laravel Breeze (React + Inertia variant)
8. **Project conventions:** Laravel skills plugin (Nuno Maduro conventions) — Actions pattern, strict models, Form Requests, Pest, Larastan level 9, Pint, Rector

## Chosen Approach

**Laravel as thin proxy (Approach A)**

Laravel acts as a proxy/cache layer between the React frontend and the Consumet API:
- All Consumet calls go through a `ConsumetService` class with caching
- Keeps Consumet URL private from the client
- Easy to swap anime sources later
- Auth, watchlist, and history live naturally in Laravel

## Features (v1)

1. **Browse/Search** — search by title, browse by genre/popularity, home page with trending/popular/recent
2. **Anime detail page** — synopsis, episode list, rating, cover art, genre tags
3. **Video player** — ArtPlayer with HLS streaming, episode navigation, auto-save progress
4. **Watchlist** — save anime with status (watching/plan_to_watch/completed/dropped)
5. **Watch history** — resume where you left off, continue watching list
6. **User auth** — login/register via Breeze

## Database Schema

- `watchlist` — user_id, anime_id (string), anime_title, anime_image, status
- `watch_history` — user_id, anime_id, episode_id (string), episode_number, progress_seconds, completed, watched_at
- Cache handled by Laravel's built-in cache system (not DB tables)

## Pages & Routes

**Public:** `/` (home), `/search`, `/genre/{genre}`, `/anime/{id}`, `/anime/{id}/watch/{episodeId}`
**Authenticated:** `/watchlist` (CRUD), `/history`, `POST /history` (progress saves)
**Auth:** Breeze standard routes

## Key Components

**Laravel:**
- `AnimeController` — home, search, genre, detail
- `StreamController` — watch page, fetches HLS links
- `WatchlistController` — CRUD
- `HistoryController` — save/retrieve progress
- `ConsumetService` — wraps Consumet HTTP calls with caching + error fallback

**React pages:** Home, Search, AnimeDetail, Watch, Watchlist, History
**Shared components:** AnimeCard, EpisodeList, VideoPlayer (ArtPlayer wrapper)

## Error Handling

- Consumet down → graceful error state, serve stale cache if available
- Streaming link broken → retry button that bypasses cache
- Source rotation → if primary provider fails, try next Consumet provider
- Rate limiting on Consumet calls to avoid upstream blocks
